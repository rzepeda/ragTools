# Contributing to RAG Factory

Thank you for your interest in contributing to RAG Factory! This guide will help you get started.

---

## Code of Conduct

We are committed to providing a welcoming and inclusive environment. Please be respectful and professional in all interactions.

---

## How to Contribute

### Reporting Bugs

- Use the [GitHub issue tracker](https://github.com/yourusername/rag-factory/issues)
- Include a clear description and steps to reproduce
- Provide system information (OS, Python version, etc.)

### Suggesting Features

- Open an issue with the "feature request" label
- Describe the use case and expected behavior
- Discuss the feature before implementing

### Contributing Code

1. Fork the repository
2. Create a feature branch (`git checkout -b feature/my-feature`)
3. Make your changes
4. Write/update tests
5. Update documentation
6. Submit a pull request
